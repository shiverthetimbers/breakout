Project Thoughts

Getting Started

I know, I know... The obligatory canvas game. "Tune in this week" - Aspiring Developers S:1 E:(9.56e11). I'm thinking the same thing. But, we gotta start somewhere.

First! To differentiate this project from others, I will not be following any guides/tutorials for this project. There are thousands of "code-with-me" follow along versions of this game out there. But I've decided to complete this with as much of my own knowledge as possible.

So, in an effor to stand out in a field of thoroughly beaten horses, this project will be my own work with reference only to language documentation or stack overflow articles. That being said, if you're an experienced developer reviewing this project... Prepare yourself. I'm sure it's not perfect.

After Completion

I hadn't used canvas before attempting this project. I realized as I read through the documentation there is literally a breakthrough game guide in the documentation... Either way, I enjoyed the project. The design portion was fun. Obviously it's a lot of fun to get something moving around the screen outside of simple CSS transitions.

I pushed myself to solve the logic issues without the tutorial. In fact, I avoided the tutorial at all costs. I referenced some of the canvas specific stuff very early on. I don't know if I solved these problems with the same logic as the tutorial. Truthfully, I doubt it. I'm sure some of my solutions are far from the most effecient. I also took some creative liberty along the way, so I'm sure it's unique in more than one way.

The most challenging aspect was actually taking the "destroyed" blocks out of play. I was able to make the block invisible easily, but the ball would still bounce off of the invisible block since the bounce logic is based on the blocks array. I tried a bunch of obscure methods on the array like array.filter() to no avail. Eventually, simply adding another nested condition in the hitBlock() function simplified the issue.

After about 3 days of serious work, I will call it complete. Is it perfect? Nope! And I may come back to make tweaks later. But it's fun, basically functional, and I built it without following a guide. If you read this far, thanks for stopping by.

Rules

The goal is to break all the blocks with the ball.

  • Move the paddle left and right with the arrows on your keyboard.
  • Position the paddle to hit the ball back towards the blocks.
  • Hitting the blocks with the ball will destroy them.
  • If you destroy all the blocks, you win!
  • If the ball gets past your paddle, game over.

Bonus Tip:There is a "sweet spot" in the center of the paddle. If the ball makes contact there, it will go straight up!
